## INTRODUCTION

We kick off streaming systems in this module. You will learn about a specific distributed event and data streaming tool: Apache Kafka. You will get some hands-on experience setting up a Kafka server, creating topics, and sending and receiving messages. Schema enforcement and evolution is even more important in streaming systems than batch systems. We will study schema enforcement best practices and tradeoffs.
 

## Learning Objectives

At the conclusion of this module, you should be able to:

* Explain the concepts and benefits of Apache Kafka
* Describe the pub/sub model and its benefits
* Understand how schema enforcement can be implemented
* Explain how schemas can be evolved in streaming systems
* Progress toward executing an end-to-end predictive modeling project using a large dataset
