## INTRODUCTION

We investigate some of the tradeoffs and challenges when storing distributed data. This includes handing partitions, or breaks in the network. When data is partitioned across servers, data skew and hot spots can arise. We will study these issues and how they can be remediated. 



## Learning Objectives

At the conclusion of this module, you should be able to:

* Define data skew and give strategies to mitigate skew
* Understand how hot spots lead to data skew
* Understand common types of network faults
* Demonstrate how to connect AWS services: IAM, EC2, S3
* Progress toward executing an end-to-end predictive modeling project using a large dataset
